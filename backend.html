Arquitectura Cliente-Servidor

Node.js, Express y APIs REST

Una guía completa sobre comunicación entre cliente y servidor

Modelo Cliente-Servidor

CLIENTE

Navegador, App Móvil, etc.

Solicita información

SERVIDOR

Node.js, Apache, etc.

Procesa y responde
Cliente
  • Inicia la comunicación
  • Envía peticiones (requests)
  • Espera respuestas
  • Procesa y muestra datos
Servidor
  • Escucha peticiones
  • Procesa la lógica de negocio
  • Accede a bases de datos
  • Envía respuestas

Puertos de Red

Los puertos son "puertas" virtuales que permiten que diferentes servicios se ejecuten en un mismo servidor

Servidor IP: 192.168.1.100

Puerto 80

HTTP (Web)

Puerto 443

HTTPS (Seguro)

Puerto 3000

Node.js (Desarrollo)

Puerto 5432

PostgreSQL

Importante: Un puerto es un número entre 0 y 65535. Los puertos del 0 al 1023 están reservados para servicios del sistema.

Peticiones HTTP

Métodos HTTP Principales:

GET

Propósito: Obtener/Leer datos del servidor

Ejemplo: Ver una lista de productos

Datos: En la URL (query params)

GET /usuarios?id=123
POST

Propósito: Crear/Enviar datos al servidor

Ejemplo: Registrar un nuevo usuario

Datos: En el cuerpo (body)

POST /usuarios {nombre: "Juan"}
Flujo de una Petición
1
Cliente envía petición
GET http://miapi.com:3000/productos
2
Servidor recibe y procesa
Express maneja la ruta y ejecuta lógica
3
Servidor responde
200 OK + JSON con datos
4
Cliente procesa respuesta
Muestra los productos en pantalla

Conexión Cliente a API

Anatomía de una URL de API
http://localhost:3000/api/productos?categoria=laptop
Protocolo
http:// o https://
Host
localhost o dominio
Puerto
3000, 80, 443
Ruta
/api/productos
Ejemplo en JavaScript (Cliente):
// Petición GET fetch('http://localhost:3000/api/productos') .then(response => response.json()) .then(data => { console.log('Productos:', data); }) .catch(error => console.error('Error:', error)); // Petición POST fetch('http://localhost:3000/api/productos', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ nombre: 'Laptop Dell', precio: 899.99 }) }) .then(response => response.json()) .then(data => console.log('Producto creado:', data));

Node.js y Express

Node.js

Entorno de ejecución de JavaScript en el servidor. Permite ejecutar código JavaScript fuera del navegador.

Express

Framework minimalista y flexible para Node.js que facilita la creación de servidores web y APIs.

Conceptos Clave:
  • Middleware: Funciones que procesan las peticiones antes de llegar a las rutas
  • Rutas: Endpoints que responden a peticiones específicas
  • Request (req): Objeto con información de la petición
  • Response (res): Objeto para enviar respuestas al cliente
Instalación:
# Inicializar proyecto npm init -y # Instalar Express npm install express

Servidor Express Completo

Servidor que responde a GET "/" y "/contacto", y POST "/mensaje"

// Importar Express const express = require('express'); const app = express(); const PORT = 3000; // Middleware para parsear JSON app.use(express.json()); // Middleware para parsear datos de formularios app.use(express.urlencoded({ extended: true })); // RUTA 1: GET / (Página principal) app.get('/', (req, res) => { res.send(` <h1>Bienvenido a mi API</h1> <p>Esta es la página principal</p> <a href="/contacto">Ir a Contacto</a> `); }); // RUTA 2: GET /contacto (Página de contacto) app.get('/contacto', (req, res) => { res.json({ mensaje: 'Página de contacto', email: 'info@miapi.com', telefono: '+598 12345678' }); }); // RUTA 3: POST /mensaje (Recibir mensaje) app.post('/mensaje', (req, res) => { // Extraer datos del body const { nombre, email, mensaje } = req.body; // Validar datos if (!nombre || !email || !mensaje) { return res.status(400).json({ error: 'Faltan datos requeridos', requeridos: ['nombre', 'email', 'mensaje'] }); } // Procesar mensaje (aquí iría lógica de guardar en BD) console.log('Mensaje recibido:', { nombre, email, mensaje }); // Responder al cliente res.status(201).json({ exito: true, mensaje: 'Mensaje recibido correctamente', datos: { nombre, email, fecha: new Date() } }); }); // Iniciar servidor app.listen(PORT, () => { console.log(`Servidor corriendo en http://localhost:${PORT}`); });

Explicación Detallada

const express = require('express');

Importa el módulo Express para crear el servidor.

const app = express();

Crea una instancia de la aplicación Express.

const PORT = 3000;

Define el puerto donde escuchará el servidor.

app.use(express.json());

Permite que Express entienda JSON en el body de las peticiones.

app.use(express.urlencoded({ extended: true }));

Permite parsear datos de formularios HTML.

app.get('/', (req, res) => {...});

req: Objeto request con info de la petición

res: Objeto response para enviar respuestas

res.send(): Envía HTML como respuesta

app.get('/contacto', (req, res) => {...});

res.json(): Envía un objeto JavaScript como JSON

Automáticamente establece el header Content-Type: application/json

app.post('/mensaje', (req, res) => {...});

req.body: Contiene los datos enviados en el POST

res.status(400): Establece el código de estado HTTP

res.status(201): 201 indica "Creado exitosamente"

app.listen(PORT, () => {...});

Inicia el servidor en el puerto especificado

El callback se ejecuta cuando el servidor está listo
