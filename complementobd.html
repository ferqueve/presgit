¿Qué es la Persistencia?

Definición

La persistencia de datos es la capacidad de guardar información de manera permanente, incluso después de que el programa o sistema se cierre.

¿Por qué es importante?
  • Mantener datos entre sesiones
  • Recuperar información histórica
  • Compartir datos entre aplicaciones
  • Garantizar integridad de la información

Tipos de Persistencia

Archivos

Almacenamiento básico en disco (txt, JSON, XML)

Bases de Datos

Sistemas estructurados y optimizados

Cloud Storage

Almacenamiento distribuido en la nube

Tipos de Bases de Datos

Relacionales (SQL)

Organizan datos en tablas con relaciones definidas

  • MySQL / MariaDB
  • PostgreSQL
  • SQLite
  • Oracle Database
No Relacionales (NoSQL)

Estructuras flexibles para diferentes necesidades

  • Documentos: MongoDB, CouchDB
  • Clave-Valor: Redis, DynamoDB
  • Grafos: Neo4j, ArangoDB
  • Columnas: Cassandra, HBase

SQLite - La Base de Datos Ligera

¿Qué es SQLite?

SQLite es una biblioteca que proporciona una base de datos SQL embebida y autónoma, sin necesidad de servidor.

Características
  • Sin configuración (zero-configuration)
  • Archivo único (.db, .sqlite)
  • Perfecta para aplicaciones móviles
  • Ideal para desarrollo y prototipado
  • Cross-platform
Ejemplo de uso
// Node.js con sqlite3 const sqlite3 = require('sqlite3').verbose(); const db = new sqlite3.Database('mibase.db'); // Crear tabla db.run(`CREATE TABLE IF NOT EXISTS usuarios ( id INTEGER PRIMARY KEY AUTOINCREMENT, nombre TEXT NOT NULL, email TEXT UNIQUE, edad INTEGER )`); // Insertar datos db.run(`INSERT INTO usuarios (nombre, email, edad) VALUES (?, ?, ?)`, ['Juan', 'juan@email.com', 25]); // Consultar db.all('SELECT * FROM usuarios', (err, rows) => { console.log(rows); });

MySQL & MariaDB

MySQL

Sistema de gestión de bases de datos relacional más popular del mundo, propiedad de Oracle.

  • Alto rendimiento
  • Escalabilidad
  • Ampliamente soportado
  • Gran comunidad

MariaDB

Fork de MySQL creado por los desarrolladores originales. Totalmente compatible con MySQL pero de código abierto.

  • 100% compatible con MySQL
  • Completamente open source
  • Más características
  • Mejor rendimiento
Ejemplo de conexión con Node.js
const mysql = require('mysql2/promise'); // Crear conexión const connection = await mysql.createConnection({ host: 'localhost', user: 'root', password: 'password', database: 'mi_aplicacion' }); // Ejecutar query const [rows] = await connection.execute( 'SELECT * FROM productos WHERE precio > ?', [100] ); console.log(rows); await connection.end();

La Familia SQL

Todas las bases de datos SQL comparten un lenguaje común: Structured Query Language

CREATE

Ejemplos de SQL
-- Crear tabla CREATE TABLE empleados ( id INT PRIMARY KEY AUTO_INCREMENT, nombre VARCHAR(100) NOT NULL, departamento VARCHAR(50), salario DECIMAL(10,2), fecha_ingreso DATE ); -- Insertar datos INSERT INTO empleados (nombre, departamento, salario, fecha_ingreso) VALUES ('Ana García', 'Desarrollo', 65000.00, '2024-01-15'); -- Consultar con JOIN SELECT e.nombre, e.salario, d.nombre_dept FROM empleados e INNER JOIN departamentos d ON e.departamento = d.id WHERE e.salario > 50000; -- Actualizar UPDATE empleados SET salario = salario * 1.10 WHERE departamento = 'Desarrollo'; -- Eliminar DELETE FROM empleados WHERE fecha_ingreso < '2020-01-01';

MongoDB - Bases de Datos Orientadas a Documentos

¿Qué es MongoDB?

MongoDB es una base de datos NoSQL orientada a documentos que almacena datos en formato JSON (BSON).

Ventajas
  • Esquema flexible: No requiere estructura fija
  • Escalabilidad horizontal: Fácil de distribuir
  • Alto rendimiento: Optimizado para lectura/escritura
  • Consultas ricas: Soporte para queries complejas
  • JSON nativo: Perfecto para aplicaciones web
Estructura de Datos
// Documento en MongoDB (JSON) { "_id": ObjectId("507f1f77bcf86cd799439011"), "nombre": "Juan Pérez", "email": "juan@email.com", "edad": 28, "direccion": { "calle": "Av. Principal 123", "ciudad": "Madrid", "codigo_postal": "28001" }, "hobbies": ["programación", "lectura", "gaming"], "activo": true, "fecha_registro": ISODate("2024-01-15T10:30:00Z") }

Nota: Los documentos pueden tener diferentes campos en la misma colección.

MongoDB - Operaciones CRUD

Ejemplos Prácticos con Node.js
const { MongoClient } = require('mongodb'); // Conexión const client = new MongoClient('mongodb://localhost:27017'); await client.connect(); const db = client.db('mi_aplicacion'); const usuarios = db.collection('usuarios'); // CREATE - Insertar documentos await usuarios.insertOne({ nombre: "María López", email: "maria@email.com", edad: 32, rol: "desarrolladora" }); await usuarios.insertMany([ { nombre: "Pedro", email: "pedro@email.com", edad: 25 }, { nombre: "Laura", email: "laura@email.com", edad: 28 } ]); // READ - Consultar documentos const usuario = await usuarios.findOne({ email: "maria@email.com" }); const mayores30 = await usuarios.find({ edad: { $gt: 30 } }).toArray(); const developers = await usuarios.find({ rol: "desarrolladora" }).sort({ edad: -1 }).limit(10).toArray(); // UPDATE - Actualizar documentos await usuarios.updateOne( { email: "maria@email.com" }, { $set: { edad: 33, ciudad: "Barcelona" } } ); await usuarios.updateMany( { rol: "desarrolladora" }, { $inc: { salario: 5000 } } // Incrementar salario ); // DELETE - Eliminar documentos await usuarios.deleteOne({ email: "pedro@email.com" }); await usuarios.deleteMany({ activo: false });

MongoDB - Consultas Avanzadas

Operadores de Consulta
// Operadores de comparación db.productos.find({ precio: { $gte: 100, $lte: 500 }, stock: { $ne: 0 } }); // Operadores lógicos db.productos.find({ $or: [ { categoria: "electrónica" }, { precio: { $lt: 50 } } ] }); // Búsqueda en arrays db.usuarios.find({ hobbies: { $in: ["programación", "gaming"] } }); // Búsqueda en subdocumentos db.usuarios.find({ "direccion.ciudad": "Madrid" }); // Expresiones regulares db.productos.find({ nombre: { $regex: /^laptop/i } });
Agregaciones
// Pipeline de agregación db.ventas.aggregate([ // Filtrar por fecha { $match: { fecha: { $gte: new Date('2024-01-01') } } }, // Agrupar por categoría { $group: { _id: "$categoria", total_ventas: { $sum: "$monto" }, cantidad: { $count: {} }, promedio: { $avg: "$monto" } } }, // Ordenar por total { $sort: { total_ventas: -1 } }, // Limitar resultados { $limit: 5 } ]); // Lookup (JOIN) db.pedidos.aggregate([ { $lookup: { from: "usuarios", localField: "usuario_id", foreignField: "_id", as: "usuario_info" } } ]);

MongoDB Compass

¿Qué es Compass?

MongoDB Compass es la interfaz gráfica oficial (GUI) para MongoDB que permite explorar y manipular datos de forma visual.

Características
  • Visualización de datos en tiempo real
  • Constructor visual de queries
  • Análisis de esquemas
  • Índices y rendimiento
  • Import/Export de datos
  • Validación de esquemas
Ideal para:

Desarrolladores que prefieren trabajar visualmente, hacer pruebas rápidas o explorar estructuras de datos sin escribir código.

Interfaz de Compass
Conexión Local
mongodb://localhost:27017
Funcionalidades
  • Explorador de bases de datos
  • Editor de documentos
  • Métricas de rendimiento
  • Shell integrada

MongoDB Atlas

¿Qué es Atlas?

MongoDB Atlas es el servicio de base de datos en la nube (DBaaS) totalmente administrado por MongoDB.

Ventajas
  • Hosting en la nube (AWS, Azure, GCP)
  • Seguridad avanzada y backups automáticos
  • Escalabilidad automática
  • Distribución global
  • Mantenimiento automatizado
  • Tier gratuito disponible (512MB)
Caso de uso:

Aplicaciones en producción que requieren alta disponibilidad, escalabilidad y seguridad sin preocuparse por la infraestructura.

Conexión a Atlas
// String de conexión de Atlas const uri = "mongodb+srv://usuario:password@cluster0.mongodb.net/mydb?retryWrites=true&w=majority"; const client = new MongoClient(uri, { useNewUrlParser: true, useUnifiedTopology: true, }); await client.connect(); console.log("Conectado a MongoDB Atlas");
Características Premium
Monitoring: Métricas en tiempo real
Alertas: Notificaciones personalizadas
Atlas Search: Búsqueda full-text
Atlas Data Lake: Análisis de big data

Otros Tipos de Bases de Datos NoSQL

Clave-Valor (Key-Value)

Ejemplos: Redis, DynamoDB, Memcached

Estructura: Pares simples de clave → valor

// Redis SET usuario:1000 "Juan Pérez" GET usuario:1000 HSET usuario:1001 nombre "María" edad 28

Uso ideal: Cache, sesiones, contadores

Bases de Grafos

Ejemplos: Neo4j, ArangoDB, Amazon Neptune

Estructura: Nodos y relaciones

// Neo4j (Cypher) CREATE (u:Usuario {nombre: 'Ana'}) CREATE (u2:Usuario {nombre: 'Luis'}) CREATE (u)-[:SIGUE]->(u2) MATCH (u:Usuario)-[:SIGUE]->(amigo) WHERE u.nombre = 'Ana' RETURN amigo.nombre

Uso ideal: Redes sociales, recomendaciones

Columnar (Wide-Column)

Ejemplos: Cassandra, HBase, ScyllaDB

Estructura: Familias de columnas

// Cassandra (CQL) CREATE TABLE usuarios ( user_id UUID PRIMARY KEY, email text, nombre text, timestamp timestamp ); INSERT INTO usuarios (user_id, email, nombre) VALUES (uuid(), 'ana@email.com', 'Ana');

Uso ideal: Big data, series temporales

Series Temporales

Ejemplos: InfluxDB, TimescaleDB, Prometheus

Estructura: Datos con timestamp

// InfluxDB temperature,location=room1 value=22.5 temperature,location=room2 value=23.1 SELECT mean(value) FROM temperature WHERE location = 'room1' AND time > now() - 1h GROUP BY time(10m)

Uso ideal: IoT, monitoreo, métricas

MongoDB con Mongoose en Express

Configuración de Conexión
// config/mongodb.js const mongoose = require('mongoose'); const connectDB = async () => { try { await mongoose.connect(process.env.MONGODB_URI, { useNewUrlParser: true, useUnifiedTopology: true, }); console.log('MongoDB conectado exitosamente'); } catch (error) { console.error('Error de conexión:', error); process.exit(1); } }; module.exports = connectDB;
Definir Modelo
// models/Usuario.js const mongoose = require('mongoose'); const usuarioSchema = new mongoose.Schema({ nombre: { type: String, required: [true, 'Nombre requerido'], trim: true }, email: { type: String, required: true, unique: true, lowercase: true }, edad: { type: Number, min: 0, max: 120 }, activo: { type: Boolean, default: true }, roles: [String], direccion: { calle: String, ciudad: String, pais: String } }, { timestamps: true // createdAt, updatedAt }); module.exports = mongoose.model('Usuario', usuarioSchema);
Rutas API
// routes/usuarios.js const express = require('express'); const router = express.Router(); const Usuario = require('../models/Usuario'); // GET todos los usuarios router.get('/', async (req, res) => { try { const usuarios = await Usuario.find(); res.json(usuarios); } catch (error) { res.status(500).json({ error: error.message }); } }); // POST crear usuario router.post('/', async (req, res) => { try { const usuario = new Usuario(req.body); await usuario.save(); res.status(201).json(usuario); } catch (error) { res.status(400).json({ error: error.message }); } }); // PUT actualizar usuario router.put('/:id', async (req, res) => { try { const usuario = await Usuario.findByIdAndUpdate( req.params.id, req.body, { new: true, runValidators: true } ); if (!usuario) { return res.status(404).json({ error: 'No encontrado' }); } res.json(usuario); } catch (error) { res.status(400).json({ error: error.message }); } }); // DELETE eliminar usuario router.delete('/:id', async (req, res) => { try { const usuario = await Usuario.findByIdAndDelete(req.params.id); if (!usuario) { return res.status(404).json({ error: 'No encontrado' }); } res.json({ mensaje: 'Eliminado exitosamente' }); } catch (error) { res.status(500).json({ error: error.message }); } }); module.exports = router;

Rutas con MySQL
// routes/productos.js const express = require('express'); const router = express.Router(); const db = require('../config/mysql'); // GET todos los productos router.get('/', async (req, res) => { try { const [rows] = await db.query( 'SELECT * FROM productos' ); res.json(rows); } catch (error) { res.status(500).json({ error: error.message }); } }); // POST crear producto router.post('/', async (req, res) => { try { const { nombre, precio, stock } = req.body; const [result] = await db.query( 'INSERT INTO productos (nombre, precio, stock) VALUES (?, ?, ?)', [nombre, precio, stock] ); res.status(201).json({ id: result.insertId, nombre, precio, stock }); } catch (error) { res.status(400).json({ error: error.message }); } }); // PUT actualizar producto router.put('/:id', async (req, res) => { try { const { nombre, precio, stock } = req.body; const [result] = await db.query( 'UPDATE productos SET nombre=?, precio=?, stock=? WHERE id=?', [nombre, precio, stock, req.params.id] ); if (result.affectedRows === 0) { return res.status(404).json({ error: 'No encontrado' }); } res.json({ id: req.params.id, nombre, precio, stock }); } catch (error) { res.status(400).json({ error: error.message }); } }); module.exports = router;

Sistema de Gestión de Tareas con MongoDB
// models/Tarea.js const mongoose = require('mongoose'); const tareaSchema = new mongoose.Schema({ titulo: { type: String, required: true }, descripcion: String, completada: { type: Boolean, default: false }, prioridad: { type: String, enum: ['baja', 'media', 'alta'], default: 'media' }, fechaLimite: Date, etiquetas: [String], usuario: { type: mongoose.Schema.Types.ObjectId, ref: 'Usuario' } }, { timestamps: true }); module.exports = mongoose.model('Tarea', tareaSchema); // routes/tareas.js const express = require('express'); const router = express.Router(); const Tarea = require('../models/Tarea'); // Obtener todas las tareas con filtros router.get('/', async (req, res) => { try { const { completada, prioridad, usuario } = req.query; const filtro = {}; if (completada !== undefined) filtro.completada = completada === 'true'; if (prioridad) filtro.prioridad = prioridad; if (usuario) filtro.usuario = usuario; const tareas = await Tarea.find(filtro) .populate('usuario', 'nombre email') .sort({ createdAt: -1 }); res.json({ total: tareas.length, tareas }); } catch (error) { res.status(500).json({ error: error.message }); } }); // Crear nueva tarea router.post('/', async (req, res) => { try { const tarea = new Tarea(req.body); await tarea.save(); await tarea.populate('usuario', 'nombre email'); res.status(201).json(tarea); } catch (error) { res.status(400).json({ error: error.message }); } }); // Marcar como completada router.patch('/:id/completar', async (req, res) => { try { const tarea = await Tarea.findByIdAndUpdate( req.params.id, { completada: true }, { new: true } ); if (!tarea) return res.status(404).json({ error: 'Tarea no encontrada' }); res.json(tarea); } catch (error) { res.status(400).json({ error: error.message }); } }); // Estadísticas de tareas router.get('/estadisticas', async (req, res) => { try { const stats = await Tarea.aggregate([ { $group: { _id: null, total: { $sum: 1 }, completadas: { $sum: { $cond: ['$completada', 1, 0] } }, pendientes: { $sum: { $cond: ['$completada', 0, 1] } } } } ]); const porPrioridad = await Tarea.aggregate([ { $group: { _id: '$prioridad', count: { $sum: 1 } } } ]); res.json({ general: stats[0], porPrioridad }); } catch (error) { res.status(500).json({ error: error.message }); } }); module.exports = router;
