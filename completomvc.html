Patrón MVC (Model-View-Controller)

¿Qué es MVC?

MVC es un patrón de arquitectura de software que separa la aplicación en tres componentes principales, cada uno con una responsabilidad específica.

Model

Datos y lógica de negocio

View

Interfaz de usuario

Controller

Lógica de control

Analogía del Restaurante
  • Model (Chef): Prepara la comida (gestiona los datos)
  • View (Plato): La presentación de la comida (lo que ve el cliente)
  • Controller (Mesero): Toma el pedido y coordina todo (gestiona las peticiones)

Flujo de una petición

// 1. Cliente hace una petición Cliente → GET /api/usuarios // 2. La ROUTE recibe y dirige la petición Route → Identifica el endpoint y llama al Controller // 3. El CONTROLLER procesa la lógica Controller → Solicita datos al Model // 4. El MODEL interactúa con la base de datos Model → Consulta la base de datos // 5. Se devuelve la respuesta Model → Controller → Route → Cliente

Routes (Rutas) - Explicación Profunda

¿Qué son las Routes?

Las rutas son el sistema de enrutamiento que mapea URLs a controladores. Actúan como un "directorio telefónico" que sabe exactamente a qué función llamar cuando llega una petición específica.

¿Por qué usar Routes?

Organización del Código

Separa la definición de URLs del código que las maneja, manteniendo todo ordenado.

Reutilización

Puedes reutilizar los mismos controladores en diferentes rutas.

Mantenibilidad

Es más fácil encontrar y modificar rutas cuando están centralizadas.

Middleware

Puedes aplicar middleware (autenticación, validación) a grupos de rutas fácilmente.

Versionado

Facilita crear diferentes versiones de tu API (v1, v2, etc.).

Tipos de Rutas

// Ruta simple app.get('/usuarios', controlador); // Ruta con parámetro app.get('/usuarios/:id', controlador); // Ruta con múltiples parámetros app.get('/usuarios/:id/posts/:postId', controlador); // Ruta con query params (ejemplo: /buscar?nombre=Juan&edad=25) app.get('/buscar', controlador); // Agrupación de rutas con prefijo const router = express.Router(); router.get('/', listarUsuarios); router.post('/', crearUsuario); app.use('/api/usuarios', router);
Ejemplo Real

Imagina una aplicación de blog. En lugar de tener todo el código mezclado, las rutas te permiten organizar:

  • /api/posts → Todo sobre publicaciones
  • /api/users → Todo sobre usuarios
  • /api/comments → Todo sobre comentarios

Controllers - Explicación Profunda

¿Qué son los Controllers?

Los controladores son funciones que contienen la lógica para manejar las peticiones. Procesan los datos de entrada, interactúan con los modelos y envían las respuestas.

¿Por qué usar Controllers?

Separación de Responsabilidades

Las rutas solo mapean URLs, los controladores manejan la lógica. Cada uno tiene su trabajo específico.

Reutilización de Código

Puedes usar el mismo controlador desde diferentes rutas o incluso diferentes aplicaciones.

Testabilidad

Es mucho más fácil hacer pruebas unitarias de funciones independientes.

Escalabilidad

A medida que tu aplicación crece, mantener la lógica organizada es crucial.

Anatomía de un Controller

// Un controlador típico tiene esta estructura: exports.nombreDelControlador = async (req, res) => { try { // 1. Extraer datos de la petición const datos = req.body; const parametros = req.params; const query = req.query; // 2. Validar datos (opcional) if (!datos.campo) { return res.status(400).json({ error: 'Campo requerido' }); } // 3. Interactuar con el modelo const resultado = await Model.operacion(datos); // 4. Procesar resultados const respuesta = procesarDatos(resultado); // 5. Enviar respuesta res.status(200).json({ success: true, data: respuesta }); } catch (error) { // 6. Manejo de errores res.status(500).json({ success: false, error: error.message }); } };
Responsabilidades de un Controller
  • ✅ Recibir y validar datos de entrada
  • ✅ Llamar a los modelos necesarios
  • ✅ Procesar y transformar datos
  • ✅ Manejar la lógica de negocio
  • ✅ Enviar respuestas al cliente
  • ✅ Manejar errores apropiadamente

Models (Modelos)

¿Qué son los Models?

Los modelos representan la estructura de los datos y contienen la lógica para interactuar con la base de datos. Definen cómo se almacenan, recuperan y manipulan los datos.

¿Por qué usar Models?

Abstracción de Datos

El resto de la aplicación no necesita saber cómo se almacenan los datos en la base de datos.

Validación

Puedes definir reglas de validación para asegurar la integridad de los datos.

Reutilización

Los mismos métodos del modelo pueden usarse desde diferentes controladores.

Cambios Centralizados

Si cambias de base de datos, solo modificas los modelos, no toda la aplicación.

Estructura de un Model

// Ejemplo con array simple (sin base de datos real) let usuarios = []; // Simulación de base de datos let nextId = 1; const Usuario = { // Obtener todos los usuarios obtenerTodos: () => { return usuarios; }, // Obtener usuario por ID obtenerPorId: (id) => { return usuarios.find(u => u.id === parseInt(id)); }, // Crear nuevo usuario crear: (datos) => { const nuevoUsuario = { id: nextId++, ...datos, fechaCreacion: new Date() }; usuarios.push(nuevoUsuario); return nuevoUsuario; }, // Actualizar usuario actualizar: (id, datos) => { const index = usuarios.findIndex(u => u.id === parseInt(id)); if (index !== -1) { usuarios[index] = { ...usuarios[index], ...datos }; return usuarios[index]; } return null; }, // Eliminar usuario eliminar: (id) => { const index = usuarios.findIndex(u => u.id === parseInt(id)); if (index !== -1) { return usuarios.splice(index, 1)[0]; } return null; } }; module.exports = Usuario;

Estructura del Proyecto

Organización de Carpetas

Vamos a crear un proyecto con una estructura profesional y organizada:

mi-api/
├── controllers/ ← Lógica de control
│ ├── usuariosController.js
│ └── productosController.js
├── models/ ← Estructura de datos
│ ├── Usuario.js
│ └── Producto.js
├── routes/ ← Definición de rutas
│ ├── usuariosRoutes.js
│ └── productosRoutes.js
├── server.js ← Archivo principal
├── package.json ← Dependencias
└── .gitignore ← Archivos a ignorar
Ventajas de esta estructura
  • Escalable: Fácil agregar nuevas funcionalidades
  • Mantenible: Cada cosa en su lugar
  • Colaborativa: Varios desarrolladores pueden trabajar simultáneamente
  • Profesional: Sigue las mejores prácticas de la industria

Flujo de Datos

Cliente hace petición a una URL

Routes identifica la ruta y llama al controlador

Controller procesa la petición

Model interactúa con los datos

Model devuelve datos al controlador

Controller formatea la respuesta

Routes envía respuesta al cliente

Paso 2: Crear los Models

1
Crear models/Usuario.js:
// models/Usuario.js // Simulamos una base de datos con un array let usuarios = [ { id: 1, nombre: 'Juan Pérez', email: 'juan@ejemplo.com', edad: 30 }, { id: 2, nombre: 'María García', email: 'maria@ejemplo.com', edad: 25 }, { id: 3, nombre: 'Pedro López', email: 'pedro@ejemplo.com', edad: 35 } ]; let nextId = 4; const Usuario = { // Obtener todos los usuarios obtenerTodos: () => { return usuarios; }, // Obtener un usuario por ID obtenerPorId: (id) => { return usuarios.find(usuario => usuario.id === parseInt(id)); }, // Crear un nuevo usuario crear: (datosUsuario) => { const nuevoUsuario = { id: nextId++, nombre: datosUsuario.nombre, email: datosUsuario.email, edad: datosUsuario.edad, fechaCreacion: new Date() }; usuarios.push(nuevoUsuario); return nuevoUsuario; }, // Actualizar un usuario existente actualizar: (id, datosActualizados) => { const index = usuarios.findIndex(u => u.id === parseInt(id)); if (index !== -1) { usuarios[index] = { ...usuarios[index], ...datosActualizados, id: usuarios[index].id // Mantener el ID original }; return usuarios[index]; } return null; }, // Eliminar un usuario eliminar: (id) => { const index = usuarios.findIndex(u => u.id === parseInt(id)); if (index !== -1) { const usuarioEliminado = usuarios.splice(index, 1)[0]; return usuarioEliminado; } return null; }, // Buscar usuarios por nombre buscarPorNombre: (nombre) => { return usuarios.filter(u => u.nombre.toLowerCase().includes(nombre.toLowerCase()) ); } }; module.exports = Usuario;
Nota: En una aplicación real, aquí usarías una base de datos como MongoDB, PostgreSQL o MySQL con un ORM como Mongoose o Sequelize.

1
Crear controllers/usuariosController.js:
// controllers/usuariosController.js const Usuario = require('../models/Usuario'); // Obtener todos los usuarios exports.obtenerTodos = (req, res) => { try { const usuarios = Usuario.obtenerTodos(); res.status(200).json({ success: true, cantidad: usuarios.length, data: usuarios }); } catch (error) { res.status(500).json({ success: false, mensaje: 'Error al obtener usuarios', error: error.message }); } }; // Obtener un usuario por ID exports.obtenerPorId = (req, res) => { try { const usuario = Usuario.obtenerPorId(req.params.id); if (!usuario) { return res.status(404).json({ success: false, mensaje: 'Usuario no encontrado' }); } res.status(200).json({ success: true, data: usuario }); } catch (error) { res.status(500).json({ success: false, mensaje: 'Error al obtener usuario', error: error.message }); } }; // Crear un nuevo usuario exports.crear = (req, res) => { try { const { nombre, email, edad } = req.body; // Validaciones básicas if (!nombre || !email) { return res.status(400).json({ success: false, mensaje: 'Nombre y email son requeridos' }); } const nuevoUsuario = Usuario.crear({ nombre, email, edad }); res.status(201).json({ success: true, mensaje: 'Usuario creado exitosamente', data: nuevoUsuario }); } catch (error) { res.status(500).json({ success: false, mensaje: 'Error al crear usuario', error: error.message }); } }; // Actualizar un usuario exports.actualizar = (req, res) => { try { const usuarioActualizado = Usuario.actualizar(req.params.id, req.body); if (!usuarioActualizado) { return res.status(404).json({ success: false, mensaje: 'Usuario no encontrado' }); } res.status(200).json({ success: true, mensaje: 'Usuario actualizado exitosamente', data: usuarioActualizado }); } catch (error) { res.status(500).json({ success: false, mensaje: 'Error al actualizar usuario', error: error.message }); } }; // Eliminar un usuario exports.eliminar = (req, res) => { try { const usuarioEliminado = Usuario.eliminar(req.params.id); if (!usuarioEliminado) { return res.status(404).json({ success: false, mensaje: 'Usuario no encontrado' }); } res.status(200).json({ success: true, mensaje: 'Usuario eliminado exitosamente', data: usuarioEliminado }); } catch (error) { res.status(500).json({ success: false, mensaje: 'Error al eliminar usuario', error: error.message }); } }; // Buscar usuarios exports.buscar = (req, res) => { try { const { nombre } = req.query; if (!nombre) { return res.status(400).json({ success: false, mensaje: 'Parámetro nombre es requerido' }); } const resultados = Usuario.buscarPorNombre(nombre); res.status(200).json({ success: true, cantidad: resultados.length, data: resultados }); } catch (error) { res.status(500).json({ success: false, mensaje: 'Error al buscar usuarios', error: error.message }); Paso 4: Crear las Routes

1
Crear routes/usuariosRoutes.js:
// routes/usuariosRoutes.js const express = require('express'); const router = express.Router(); const usuariosController = require('../controllers/usuariosController'); // Ruta base: /api/usuarios // GET /api/usuarios - Obtener todos los usuarios router.get('/', usuariosController.obtenerTodos); // GET /api/usuarios/buscar?nombre=Juan - Buscar usuarios router.get('/buscar', usuariosController.buscar); // GET /api/usuarios/:id - Obtener un usuario específico router.get('/:id', usuariosController.obtenerPorId); // POST /api/usuarios - Crear un nuevo usuario router.post('/', usuariosController.crear); // PUT /api/usuarios/:id - Actualizar un usuario router.put('/:id', usuariosController.actualizar); // DELETE /api/usuarios/:id - Eliminar un usuario router.delete('/:id', usuariosController.eliminar); module.exports = router;

Paso 5: Crear el Servidor

1
Crear server.js en la raíz del proyecto:
// server.js const express = require('express'); const app = express(); const PORT = process.env.PORT || 3000; // Importar rutas const usuariosRoutes = require('./routes/usuariosRoutes'); // Middlewares // Permite recibir JSON en las peticiones app.use(express.json()); // Permite recibir datos de formularios app.use(express.urlencoded({ extended: true })); // Ruta de inicio app.get('/', (req, res) => { res.json({ mensaje: '¡Bienvenido a mi API!', version: '1.0.0', endpoints: { usuarios: '/api/usuarios' } }); }); // Usar las rutas app.use('/api/usuarios', usuariosRoutes); // Ruta para manejar errores 404 app.use((req, res) => { res.status(404).json({ success: false, mensaje: 'Ruta no encontrada' }); }); // Iniciar el servidor app.listen(PORT, () => { console.log(`🚀 Servidor corriendo en http://localhost:${PORT}`); console.log(`📚 API Usuarios: http://localhost:${PORT}/api/usuarios`); });
2
Iniciar el servidor:
# Modo desarrollo (con nodemon) npm run dev # O modo producción npm start
¡Servidor listo!

Tu API ahora está corriendo en http://localhost:3000

Paso 6: Probar la API

Pruebas con cURL o Postman

1. Obtener todos los usuarios
curl http://localhost:3000/api/usuarios
2. Obtener un usuario específico
curl http://localhost:3000/api/usuarios/1
3. Crear un nuevo usuario (POST)
curl -X POST http://localhost:3000/api/usuarios \ -H "Content-Type: application/json" \ -d '{"nombre":"Ana Martínez","email":"ana@ejemplo.com","edad":28}'
4. Actualizar un usuario (PUT)
curl -X PUT http://localhost:3000/api/usuarios/1 \ -H "Content-Type: application/json" \ -d '{"edad":31}'
5. Eliminar un usuario (DELETE)
curl -X DELETE http://localhost:3000/api/usuarios/3
6. Buscar usuarios
curl http://localhost:3000/api/usuarios/buscar?nombre=Juan
Herramientas Recomendadas
  • Postman: Interfaz gráfica para probar APIs
  • Insomnia: Alternativa a Postman
