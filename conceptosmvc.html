Conceptos de Desarrollo Web Backend

Endpoints, GET, POST, Routes y Controllers

¿Qué es un Endpoint?

Definición

Un endpoint es una URL específica de una API donde se puede acceder a un recurso o servicio. Es como una "puerta de entrada" para comunicarse con el servidor.

Analogía

Imagina que el endpoint es como una dirección postal específica. Si quieres enviar una carta a alguien, necesitas saber su dirección exacta. De la misma forma, si quieres obtener datos del servidor, necesitas saber el endpoint correcto.

¿Para qué se usa?

  • Para acceder a recursos específicos en el servidor
  • Para realizar operaciones como obtener, crear, actualizar o eliminar datos
  • Para estructurar y organizar las funcionalidades de una API

Ejemplos de Endpoints

https://api.ejemplo.com/usuarios ← Obtener todos los usuarios https://api.ejemplo.com/usuarios/123 ← Obtener usuario con ID 123 https://api.ejemplo.com/productos ← Obtener todos los productos https://api.ejemplo.com/pedidos/nuevo ← Crear un nuevo pedido

Método GET

Definición

GET es un método HTTP utilizado para solicitar y obtener datos de un servidor sin modificarlos. Es una operación de solo lectura.

Analogía

Es como ir a una biblioteca y consultar un libro. Puedes leerlo y ver su contenido, pero no lo modificas ni te lo llevas. Solo obtienes la información.

Características

  • Solo obtiene datos, no los modifica
  • Los parámetros se envían en la URL
  • Se puede guardar en el historial del navegador
  • Idempotente: repetir la misma petición da el mismo resultado

Ejemplo Práctico Paso a Paso

1
Definir la URL del endpoint:
GET https://api.tienda.com/productos
2
Agregar parámetros opcionales en la URL:
GET https://api.tienda.com/productos?categoria=electronicos&precio_max=1000
3
El servidor procesa la petición y devuelve los datos:
{ "productos": [ { "id": 1, "nombre": "Laptop HP", "precio": 800, "categoria": "electronicos" }, { "id": 2, "nombre": "Mouse Logitech", "precio": 25, "categoria": "electronicos" } ] }

Método POST

Definición

POST es un método HTTP utilizado para enviar datos al servidor para crear o procesar un nuevo recurso. Modifica el estado del servidor.

Analogía

Es como llenar un formulario de registro y enviarlo por correo. Estás creando un nuevo registro con tu información que el servidor guardará en su base de datos.

Características

  • Crea nuevos recursos en el servidor
  • Los datos se envían en el cuerpo (body) de la petición
  • No aparece en el historial del navegador
  • Más seguro para enviar información sensible
  • No es idempotente: repetir la petición puede crear múltiples recursos

Ejemplo Práctico Paso a Paso

1
Definir el endpoint para crear un recurso:
POST https://api.tienda.com/usuarios/registro
2
Preparar los datos a enviar en formato JSON:
{ "nombre": "María García", "email": "maria@ejemplo.com", "password": "miPassword123", "edad": 28 }
3
El servidor procesa los datos y crea el nuevo usuario:
{ "success": true, "mensaje": "Usuario creado exitosamente", "usuario": { "id": 456, "nombre": "María García", "email": "maria@ejemplo.com", "fecha_registro": "2025-10-27" } }

Routes (Rutas)

Definición

Las rutas son definiciones que mapean URLs específicas a funciones o controladores que deben ejecutarse cuando se accede a esas URLs. Son como un "mapa" que le dice a la aplicación qué hacer cuando recibe una petición.

Analogía

Las rutas son como las señales de tráfico en una ciudad. Te indican qué camino tomar para llegar a tu destino. Cuando llegas a una dirección (URL), la ruta te dirige al lugar correcto (función) que maneja tu solicitud.

¿Para qué se usan?

  • Para organizar los endpoints de tu aplicación
  • Para conectar URLs con las funciones que las manejan
  • Para definir qué métodos HTTP (GET, POST, etc.) se aceptan
  • Para estructurar la navegación de tu API

Ejemplo Práctico con Express.js (Node.js)

1
Importar Express y crear la aplicación:
const express = require('express'); const app = express();
2
Definir rutas básicas:
// Ruta para la página principal app.get('/', (req, res) => { res.send('¡Bienvenido a mi API!'); }); // Ruta para obtener todos los usuarios app.get('/usuarios', (req, res) => { res.json({ usuarios: ['Ana', 'Juan', 'Pedro'] }); }); // Ruta para obtener un usuario específico app.get('/usuarios/:id', (req, res) => { const id = req.params.id; res.json({ id: id, nombre: 'Usuario ' + id }); }); // Ruta para crear un nuevo usuario app.post('/usuarios', (req, res) => { res.json({ mensaje: 'Usuario creado' }); });
3
Iniciar el servidor:
app.listen(3000, () => { console.log('Servidor corriendo en puerto 3000'); });
Nota: Las rutas pueden agruparse por funcionalidad usando routers para mantener el código organizado.

Controllers (Controladores)

Definición

Los controladores son funciones o clases que contienen la lógica de negocio de tu aplicación. Procesan las peticiones, interactúan con la base de datos y devuelven las respuestas. Separan la lógica de las rutas.

Analogía

Un controlador es como el chef de un restaurante. Las rutas son como el mesero que toma tu pedido, pero el chef (controlador) es quien realmente prepara la comida (procesa la lógica) y la entrega lista para servir.

¿Para qué se usan?

  • Para separar la lógica de negocio de las rutas
  • Para mantener el código organizado y reutilizable
  • Para facilitar el mantenimiento y pruebas
  • Para aplicar el principio de responsabilidad única

Ejemplo Práctico Paso a Paso

1
Crear el archivo del controlador (usuariosController.js):
// Simulamos una base de datos const usuarios = [ { id: 1, nombre: 'Ana', email: 'ana@ejemplo.com' }, { id: 2, nombre: 'Juan', email: 'juan@ejemplo.com' } ]; // Controlador para obtener todos los usuarios exports.obtenerUsuarios = (req, res) => { res.json({ success: true, data: usuarios }); }; // Controlador para obtener un usuario específico exports.obtenerUsuarioPorId = (req, res) => { const id = parseInt(req.params.id); const usuario = usuarios.find(u => u.id === id); if (usuario) { res.json({ success: true, data: usuario }); } else { res.status(404).json({ success: false, mensaje: 'Usuario no encontrado' }); } }; // Controlador para crear un nuevo usuario exports.crearUsuario = (req, res) => { const nuevoUsuario = { id: usuarios.length + 1, nombre: req.body.nombre, email: req.body.email }; usuarios.push(nuevoUsuario); res.status(201).json({ success: true, mensaje: 'Usuario creado', data: nuevoUsuario }); };
2
Conectar el controlador con las rutas:
const express = require('express'); const router = express.Router(); const usuariosController = require('./usuariosController'); // Definir rutas usando los controladores router.get('/usuarios', usuariosController.obtenerUsuarios); router.get('/usuarios/:id', usuariosController.obtenerUsuarioPorId); router.post('/usuarios', usuariosController.crearUsuario); module.exports = router;
3
Usar las rutas en la aplicación principal:
const express = require('express'); const app = express(); const usuariosRoutes = require('./usuariosRoutes'); app.use(express.json()); app.use('/api', usuariosRoutes); app.listen(3000);
Ventaja: Esta estructura hace que el código sea más mantenible, testeable y escalable.

Resumen de Conceptos

Endpoint

Qué es: URL específica de una API

Para qué: Acceder a recursos del servidor

/api/usuarios

GET

Qué es: Método HTTP para obtener datos

Para qué: Leer información sin modificarla

GET /api/productos

POST

Qué es: Método HTTP para enviar datos

Para qué: Crear nuevos recursos

POST /api/usuarios

Routes

Qué son: Mapeo de URLs a funciones

Para qué: Organizar los endpoints

app.get('/users', fn)

Controllers

Qué son: Funciones con la lógica de negocio

Para qué: Procesar peticiones y separar responsabilidades

exports.getUsers = (req, res) => { /* lógica */ }
Flujo Completo

Cliente hace petición → Endpoint recibe la petición → Route identifica la ruta → Controller procesa la lógica → Respuesta regresa al cliente
