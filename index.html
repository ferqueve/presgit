¿Qué es Git?

Sistema de Control de Versiones

Git es una herramienta que te permite guardar el historial de cambios de tu código, como una máquina del tiempo para tus archivos.

¿Por qué usar Git?

  • Historial completo de cambios
  • Trabajo en equipo eficiente
  • Experimentar sin miedo
  • Revertir cambios fácilmente

Analogía Simple

Imagina Git como Google Docs para programadores: múltiples personas pueden trabajar en el mismo proyecto, ver el historial de cambios, y recuperar versiones anteriores.

¿Qué es un Repositorio?

Tu Proyecto con Superpoderes

Un repositorio (o "repo") es una carpeta especial donde Git guarda tu proyecto y todo su historial.

Tipos de Repositorios

  • Local: En tu computadora
  • Remoto: En servicios como GitHub, GitLab

Crear un Repositorio

$ git init
# Inicializa un nuevo repositorio
Un repositorio contiene todos los archivos de tu proyecto más la carpeta oculta .git con el historial.

Staging Area

La Sala de Espera de tus Cambios

El staging area es como una zona de preparación donde seleccionas qué cambios quieres guardar en el próximo commit.

$ git add archivo.txt
# Agrega archivo.txt al staging area

$ git add .
# Agrega todos los cambios al staging area

¿Qué es un Commit?

Una Fotografía de tu Código

Un commit es como tomar una foto del estado actual de tu proyecto. Cada commit tiene un mensaje que describe qué cambios hiciste.

Características de un Commit

  • ID único (hash)
  • Autor
  • Fecha y hora
  • Mensaje descriptivo

Hacer un Commit

$ git commit -m "Agregué login de usuarios"
# Crea un commit con mensaje
Tip: Escribe mensajes claros y descriptivos

Branches (Ramas)

Universos Paralelos de tu Código

Las branches te permiten crear versiones alternativas de tu proyecto para experimentar sin afectar la versión principal.

Crear una Branch
$ git branch nueva-funcionalidad
$ git checkout nueva-funcionalidad
O en un solo comando
$ git checkout -b nueva-funcionalidad

Push y Pull

Git Push

Envía tus commits locales al repositorio remoto (GitHub, GitLab, etc.)

$ git push origin main
# Sube cambios a la rama main

Git Pull

Descarga los cambios del repositorio remoto a tu repositorio local

$ git pull origin main
# Descarga cambios de main
Flujo típico: Pull (actualizar) → Hacer cambios → Add → Commit → Push (compartir)
11 / 15

Merge (Fusionar)

Uniendo Branches

Merge combina los cambios de una branch con otra, típicamente para integrar una nueva funcionalidad a la rama principal.

Proceso de Merge

  1. Cambiar a la rama destino (ej: main)
  2. Ejecutar el comando merge
  3. Resolver conflictos si existen
  4. Confirmar el merge

Comandos

$ git checkout main
# Cambiar a main

$ git merge nueva-funcionalidad
# Fusionar la branch
Conflictos: Ocurren cuando Git no puede fusionar automáticamente. Debes resolverlos manualmente.

Trabajo Colaborativo con Git

Mejores Prácticas para Equipos

Flujo de Trabajo Recomendado

  • Crear branch para cada feature
  • Commits frecuentes y descriptivos
  • Pull requests para revisión
  • Code reviews antes de merge
  • Mantener main/master estable

Plataformas Populares

GitHub

La más popular para proyectos open source

GitLab

Completa con CI/CD integrado

Bitbucket

Popular en empresas con Atlassian
