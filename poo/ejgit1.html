¿Qué es Git?

Git es un sistema de control de versiones distribuido. Registra cada cambio que haces en tu código, permitiéndote:

  • Guardar historial — cada versión queda registrada y es recuperable
  • Trabajar en equipo — múltiples personas editan el mismo proyecto sin pisarse
  • Experimentar sin miedo — creás ramas, probás y las descartás si no funcionan
  • Sincronizar con un servidor — GitHub, GitLab, Bitbucket guardan tu código en la nube
CONCEPTO CLAVE Cada desarrollador tiene una copia completa del repositorio en su máquina. Si el servidor falla, ¡el historial está seguro en todos los equipos del equipo!

Los 8 Comandos del Desafío

   terminal — bash
git clone
Descarga un repositorio remoto y crea una copia local completa con todo el historial.
git checkout
Cambia entre ramas o crea una nueva con el flag -b. También restaura archivos.
git pull
Descarga los últimos cambios del repositorio remoto y los integra en la rama actual.
git add
Agrega archivos al staging area (área de preparación) para incluirlos en el próximo commit.
git commit
Guarda una instantánea de los cambios del staging area con un mensaje descriptivo.
git stash
Guarda temporalmente cambios sin commitear. Útil cuando necesitás cambiar de rama sin perder trabajo. Se recupera con git stash pop.
git merge
Combina los cambios de una rama en la rama actual. Puede generar conflictos que hay que resolver manualmente.
git push
Envía los commits locales al repositorio remoto para que los demás puedan verlos y descargarlos.
TIP El flujo básico es siempre: add → commit → push. Primero preparás, luego registrás, luego compartís.

¿Qué son las Ramas (Branches)?

Una rama es una línea de desarrollo independiente. Imaginala como una copia paralela del proyecto donde podés trabajar sin afectar el código principal.

main / master

La rama principal. Contiene el código estable y "oficial" del proyecto.

feature branches

Ramas de trabajo donde cada estudiante desarrolla su parte sin interferir con el resto.

VENTAJA Si una rama "sale mal", no afecta a main. Simplemente la descartás y empezás de nuevo.

Guardar Temporalmente con Stash

Situación: estás trabajando en cambios a mitad, y necesitás "pausar" sin hacer commit. git stash los guarda en una pila temporal.

1. Agregar este comentario antes de tu clase en Program.cs:

// Esta clase implementa la operación <nombre-operación>
public class Resta
{
  public static int Restar(...) { ... }
}

2. Guardar temporalmente sin commitear:

$ git stash

3. Recuperar los cambios guardados:

$ git stash pop
$ git push origin suma Diapositiva 12 / 15

Enviar Cambios al Repositorio Remoto

Primero, commitear el comentario del paso anterior, luego subir la rama al servidor:

$ git commit -m "Comentario agregado"
$ git push origin <nombre-rama>

Obtener Cambios y Hacer Merge

De a un estudiante a la vez, cada uno ejecuta:

$ git pull
$ git merge <nombre-rama>
$ git push
SIN CONFLICTOS Fast-forward ... 1 file changed, 9 insertions(+)
CON CONFLICTOS CONFLICT (content): Merge conflict in src/Program/Program.cs
→ Resolvé manualmente, luego: git add . && git commit && git push

Código en conflicto — qué verás en el editor:

  Console.WriteLine(...);
<<<<<<< HEAD
  // código en main actual
=======
  // código de tu rama
>>>>>>> suma
CÓMO RESOLVER En este ejercicio, ambos bloques deben quedar. Solo borrá los marcadores <<<, === y >>>. En otros casos, elegís qué código conservar.

Resumen — Lo que Aprendiste

Control de Versiones

Git registra cada cambio, permite volver al pasado y trabaja de forma distribuida.

Ramas y Trabajo Paralelo

Cada integrante trabajó en su rama sin pisar el código de los demás.

Integración y Conflictos

Merge combina el trabajo del equipo; los conflictos se resuelven manualmente.
