Implementación explícita de interfaces

Cómo resolver dos interfaces que tienen el mismo método, pero deben comportarse distinto.

Ejemplo principal

Una estación meteorológica entrega la temperatura en Celsius y Fahrenheit usando el mismo nombre de método: GetTemperature().

El problema

Dos interfaces tienen un método con la misma firma:

ICelsius.GetTemperature() IFahrenheit.GetTemperature() 
interface ICelsius 
{
    double GetTemperature();
}

interface IFahrenheit 
{
    double GetTemperature();
}
La clase necesita responder diferente dependiendo de la interfaz usada.
Idea central

El método queda “oculto” detrás de cada interfaz

Clase

WheatherStation guarda la temperatura base en Celsius.

Interfaces

Cada interfaz expone su propia versión de GetTemperature().

Sí funciona: ICelsius c = w; · IFahrenheit f = w;

No funciona: w.GetTemperature()

Solución

Código correcto con implementación explícita

Observá que los métodos no tienen public. Se escriben con el nombre de la interfaz delante.

using System;

interface ICelsius 
{
    double GetTemperature();
}

interface IFahrenheit 
{
    double GetTemperature();
}

public class WheatherStation : ICelsius, IFahrenheit
{
    private double celsius;

    public WheatherStation(double celsius)
    {
        this.celsius = celsius;
    }

    double ICelsius.GetTemperature() => celsius;

    double IFahrenheit.GetTemperature() => celsius * 9.0 / 5.0 + 32;
}
class Program
{
    static void Main()
    {
        WheatherStation w = new WheatherStation(30);

        ICelsius c = w;
        Console.WriteLine($"Celsius: {c.GetTemperature()}");

        IFahrenheit f = w;
        Console.WriteLine($"Fahrenheit: {f.GetTemperature()}");
    }
}

Forma clásica con Main

Es la opción más segura para evitar el error de top-level statements en VS Code.

Salida esperada
Celsius: 30
Fahrenheit: 86

El error de VS Code

“Las instrucciones de nivel superior deben preceder a las declaraciones de espacio de nombres y de tipos.”

Causa

Pusiste código ejecutable después de declarar interfaces o clases usando top-level statements.

Solución recomendada

Usá class Program con static void Main(), especialmente si estás aprendiendo o haciendo ejercicios académicos.

Resumen final

Qué tenés que recordar

Misma firma

Dos interfaces pueden tener el mismo método: GetTemperature().

Método oculto

Con implementación explícita, el método solo se llama desde la interfaz.

Ideal para exámenes

Usá Main() para evitar errores de orden con top-level statements.
