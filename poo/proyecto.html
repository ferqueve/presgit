Idea central

El motor debe estar desacoplado

El recomendador no debería depender directamente de una clase llamada Cancion.

Diseño limitado
public class Recomendador
{
    public List<Cancion> Recomendar(List<Cancion> canciones)
    {
        // Solo sirve para canciones
    }
}
Diseño reutilizable
public class Recomendador
{
    public List<IRecomendable> Recomendar(List<IRecomendable> items)
    {
        // Sirve para canciones, películas, productos...
    }
}

¿Por qué separar?

Porque permite que el proyecto crezca sin mezclar todo en un solo archivo.

  • Models: clases que representan datos.
  • Interfaces: contratos que otras clases deben cumplir.
  • Services: clases que realizan acciones o procesos.
  • Program.cs: punto de entrada para probar.
Interfaz

Paso 3: crear IRecomendable

La interfaz define qué debe tener cualquier cosa que pueda ser recomendada. 

namespace RecomendadorBot.Interfaces;

public interface IRecomendable
{
    int Id { get; }
    string Nombre { get; }
    List<string> Atributos { get; }
}
Una canción, una película o un producto podrían implementar esta interfaz.

Recomendador

Paso 6: crear Recomendador sin LINQ

Primero escribimos la lógica paso a paso.

using RecomendadorBot.Interfaces;
using RecomendadorBot.Models;

namespace RecomendadorBot.Services;

public class Recomendador
{
    public List<IRecomendable> RecomendarPorPreferencias(
        Usuario usuario,
        List<IRecomendable> catalogo)
    {
        List<IRecomendable> resultado = new List<IRecomendable>();

        foreach (IRecomendable item in catalogo)
        {
            bool yaConsumido = false;

            foreach (int id in usuario.HistorialIds)
            {
                if (id == item.Id)
                {
                    yaConsumido = true;
                    break;
                }
            }

            if (yaConsumido)
            {
                continue;
            }

            bool coincide = false;

            foreach (string atributo in item.Atributos)
            {
                foreach (string preferencia in usuario.Preferencias)
                {
                    if (atributo == preferencia)
                    {
                        coincide = true;
                        break;
                    }
                }

                if (coincide)
                {
                    break;
                }
            }

            if (coincide)
            {
                resultado.Add(item);
            }
        }

        return resultado;
    }
}
Prueba

Paso 7: probar en Program.cs

using RecomendadorBot.Interfaces;
using RecomendadorBot.Models;
using RecomendadorBot.Services;

Usuario usuario = new Usuario
{
    Id = 1,
    Nombre = "Ana",
    Preferencias = new List<string> { "rock", "pop" },
    HistorialIds = new List<int> { 1 }
};

List<IRecomendable> canciones = new List<IRecomendable>
{
    new Cancion { Id = 1, Nombre = "Ya escuchada", Artista = "Banda A", Atributos = new() { "rock" } },
    new Cancion { Id = 2, Nombre = "Rock nuevo", Artista = "Banda B", Atributos = new() { "rock" } },
    new Cancion { Id = 3, Nombre = "Clásica", Artista = "Banda C", Atributos = new() { "clasica" } },
    new Cancion { Id = 4, Nombre = "Pop moderno", Artista = "Banda D", Atributos = new() { "pop" } }
};

Recomendador recomendador = new Recomendador();

List<IRecomendable> recomendaciones =
    recomendador.RecomendarPorPreferencias(usuario, canciones);

Console.WriteLine($"Recomendaciones para {usuario.Nombre}:");

foreach (IRecomendable item in recomendaciones)
{
    Console.WriteLine($"- {item.Nombre}");
}
Resultado esperado

Al ejecutar:

dotnet run

Salida esperada:

Recomendaciones para Ana:
- Rock nuevo
- Pop moderno
La canción "Ya escuchada" no aparece porque ya está en el historial.
