Contratos entre Objetos

Los objetos colaboran entre sí enviando y recibiendo mensajes. Para que esta colaboración sea eficaz, existe un contrato entre el emisor y el receptor.

Emisor

Obligación: enviar solo mensajes dentro del contrato.
Derecho: solicitar cualquier operación del contrato.

Receptor

Obligación: tener un método para cada operación del contrato.
Derecho: implementar solo lo que el contrato establece.

Tipo

Un tipo es un conjunto de operaciones que determina los mensajes que pueden ser enviados a los objetos de ese tipo. Es la realización del contrato.

Analogía: cuando decimos que algo es un auto, sabemos exactamente qué podemos pedirle (encender, apagar, desplazarse) y qué no (recitar una biografía).

Firma de una Operación

Firma

La firma de una operación está compuesta por el nombre, los parámetros y sus tipos, y el tipo del resultado. No hay firmas duplicadas dentro de un mismo tipo.

Ejemplo en C#

// Firma: nombre="Calcular", parámetros=(double base, double altura),
//         resultado=double
double Calcular(double base, double altura);

// Firma diferente (distinto tipo de resultado)
string Calcular(double base, double altura);

// Firma diferente (distinto número de parámetros)
double Calcular(double radio);

Componentes de la firma

  • Nombre de la operación
  • Parámetros (cantidad y tipos)
  • Tipo del resultado

Restricción importante

Dentro de un mismo tipo, no puede haber dos operaciones con la misma firma. Sí puede haber operaciones con el mismo nombre pero firmas distintas (sobrecarga).

Clases que Implementan Tipos

Clases implementan tipos

La clase de un objeto implementa un tipo si tiene un método para cada una de las operaciones incluidas en el tipo, con firmas exactamente coincidentes.

Diferencia: Tipo vs. Clase

Tipo

Solo especifica qué puede hacer un objeto. Sin implementación, sin estado.

Clase

Especifica qué y cómo. Incluye código (métodos) y estado (atributos).

Consecuencia clave

No se puede instanciar un tipo directamente, porque no define comportamiento ni atributos. Para obtener un objeto de tipo A, necesitamos una clase que lo implemente.

Un objeto, más de un tipo

Un objeto puede tener más de un tipo si su clase implementa múltiples tipos.

Un tipo, más de un objeto

Objetos de clases distintas pueden compartir el mismo tipo.

Subtipo y Supertipo

Subtipo

Un tipo A es subtipo de B si el conjunto de operaciones de A contiene el de B.

A es más concreto que B.

Supertipo

Un tipo A es supertipo de B si el conjunto de operaciones de A está contenido en el de B.

A es más abstracto que B.

Principio de Sustitución

En cualquier lugar donde se espera un objeto de un tipo, puede aparecer un objeto de un subtipo sin que el comportamiento del programa cambie.

Polimorfismo y Sobrecarga

Polimorfismo

Permite que una misma definición pueda ser usada con diferentes tipos. Una operación es polimórfica cuando puede actuar sobre un tipo y todos sus subtipos.

// Polimorfismo: el mismo código funciona con Car y con Actor
ILikeable likeable;

likeable = new Car("Citroën C4");   // Car implementa ILikeable
likeable.Like();

likeable = new Actor("John Travolta"); // Actor también implementa ILikeable
likeable.Like();

// El mismo método Like() funciona para ambos — eso es polimorfismo
Sobrecarga

Una operación está sobrecargada cuando hay más de una con el mismo nombre pero con firmas diferentes (distintos parámetros, tipos o resultado).

// Sobrecarga del método Add en Library
public void Add(Movie movie) { ... }  // firma 1
public void Add(Song song)   { ... }  // firma 2 — mismo nombre, distinto parámetro

Interfaces en C#

C# provee la palabra clave interface para declarar tipos de forma explícita, separada de las clases.

IElectric

public interface IElectric
{
    Boolean IsOn { get; }
    void TurnOn();
    void TurnOff();
}

ILikeable

public interface ILikeable
{
    Int32 Likes { get; }
    void Like();
    void Unlike();
}

Car implementa ambas interfaces

public class Car : IElectric, ILikeable
{
    private String model;
    private Boolean isOn;
    private Int32 likes;

    public Boolean IsOn => this.isOn;
    public Int32 Likes => this.likes;
    public void TurnOn()  { this.isOn = true; }
    public void TurnOff() { this.isOn = false; }
    public void Like()    { this.likes++; }
    public void Unlike()  { if (this.likes > 0) this.likes--; }
}
Por convención, los nombres de interfaces en C# empiezan con I mayúscula.

Principio de Sustitución de Liskov

Liskov Substitution Principle (LSP)

"Si para cada objeto O de tipo S existe un objeto O' de tipo T tal que para todos los programas P definidos en términos de T, el comportamiento de P permanece sin cambios cuando O es sustituido por O', entonces S es subtipo de T."

— Bárbara Liskov, 1987

// T = ILikeable, S = Car y S = Actor
ILikeable likeable;

// Puedo sustituir Car por Actor (o viceversa) sin cambiar el comportamiento
likeable = new Car("Citroën");
likeable.Like();
Console.WriteLine(likeable.Likes); // 1

likeable = new Actor("John Travolta");
likeable.Like();
Console.WriteLine(likeable.Likes); // 1 (mismo comportamiento)

Permite

Una página web que muestra likes puede usar cualquier ILikeable indistintamente: Car o Actor. El código no cambia.

No confundir con herencia

El LSP aplica a tipos, no solo a jerarquías de clases. Car y Actor no se heredan entre sí, pero comparten un subtipo común.

Herencia de Interfaces

Una interfaz puede heredar de una o varias interfaces. Una clase que implementa la interfaz derivada debe implementar todos los miembros, tanto de la derivada como de sus bases.

interface IDrawable
{
    void Draw();
}

interface IShape : IDrawable
{
    double Area { get; }
}

public class Circle(double radius) : IShape
{
    public double Area =>
        Math.PI * radius * radius;

    public void Draw() =>
        Console.WriteLine($"Círculo área {Area:F2}");
}

Interface vs. Clase Abstracta

Usá interfaz cuando el tipo se implementará en jerarquías no relacionadas o se necesitan varios contratos.
Usá clase abstracta cuando los tipos relacionados comparten estado (campos) o constructores.

Implementación Explícita de Interfaces

Cuando dos interfaces declaran un miembro con el mismo nombre, la implementación explícita vincula el miembro directamente al nombre de la interfaz. Solo es accesible a través de una variable del tipo de interfaz.

interface IMetric  { double GetDistance(); } // en metros
interface IImperial { double GetDistance(); } // en pies

public class Runway(double meters) : IMetric, IImperial
{
    // Implementación explícita — nombre de interfaz + punto
    double IMetric.GetDistance()   => meters;
    double IImperial.GetDistance() => meters * 3.28084;
}

// Uso:
Runway r = new Runway(100);
IMetric m = r;   Console.WriteLine(m.GetDistance());   // 100
IImperial i = r; Console.WriteLine(i.GetDistance());   // 328.084

Casos de uso

  • Dos interfaces con el mismo nombre de miembro
  • Mantener limpia la API pública de la clase
  • Interfaces internas con tipos no públicos

Restricción

Un miembro implementado explícitamente no es accesible directamente a través de una variable del tipo de la clase — solo mediante una variable del tipo de interfaz.

Tipos Genéricos

Genéricos

Mecanismo para implementar declarativamente relaciones de generalización entre un tipo base y uno o más tipos mediante el uso de tipos parámetro.

Sin genéricos — problema

// ArrayList acepta cualquier objeto → typecast necesario
IList songs = new ArrayList();
songs.Add("Bohemian Rhapsody");
songs.Add(42); // ¡Error silencioso!

string s = (string)songs[0]; // typecast requerido

Con genéricos — solución

// List<T> controla el tipo en compilación
List<string> songs = new List<string>();
songs.Add("Bohemian Rhapsody");
// songs.Add(42); → error de compilación ✓

string s = songs[0]; // sin typecast

GenericContainer<T> — ejemplo completo

public class GenericContainer<T>
{
    private IList<T> items = new List<T>();

    public void Add(T item) { items.Add(item); }
    public IEnumerator<T> GetEnumerator() => items.GetEnumerator();
}

// Uso:
GenericContainer<Movie> movies = new GenericContainer<Movie>();
GenericContainer<Song>  songs  = new GenericContainer<Song>();
// Una sola clase, dos usos seguros en tiempo de compilación

Resumen — Interfaces en C#

Contrato

Define métodos, propiedades, eventos e indexadores. No puede declarar campos ni constructores de instancia.

Nomenclatura

Los nombres de interfaz empiezan convencionalmente con I mayúscula: IEnumerable, IComparable, IDisposable.

Sin instancia

No se puede instanciar una interfaz directamente. Se necesita una clase que la implemente.

Múltiples interfaces

Una clase puede implementar varias interfaces y heredar de una clase base simultáneamente. Esto suple la falta de herencia múltiple de clases en C#.

Declaración de tipos

Los tipos pueden declararse explícitamente (con interface) o implícitamente (las clases siempre declaran un tipo implícito).

Formas Geométricas — IShape

Definí la interfaz IShape con las propiedades Area y Perimeter (ambas double, solo lectura). Implementá las clases Circle (radio), Rectangle (ancho y alto) y Triangle (tres lados, usá fórmula de Herón). Creá una lista de IShape con instancias de cada tipo, iterá e imprimí área y perímetro.

public interface IShape
{
    double Area      { get; }
    double Perimeter { get; }
    string Name      { get; }
}

public class Circle(double radius) : IShape
{
    public double Area      => Math.PI * radius * radius;
    public double Perimeter => 2 * Math.PI * radius;
    public string Name      => "Círculo";
}

public class Rectangle(double width, double height) : IShape
{
    public double Area      => width * height;
    public double Perimeter => 2 * (width + height);
    public string Name      => "Rectángulo";
}

public class Triangle(double a, double b, double c) : IShape
{
    public double Perimeter => a + b + c;
    public double Area {
        get {
            double s = Perimeter / 2;
            return Math.Sqrt(s*(s-a)*(s-b)*(s-c));
        }
    }
    public string Name => "Triángulo";
}

// Program.cs
List<IShape> shapes = [
    new Circle(5),
    new Rectangle(4, 6),
    new Triangle(3, 4, 5)
];

foreach (IShape s in shapes)
    Console.WriteLine($"{s.Name}: Área={s.Area:F2}, Perímetro={s.Perimeter:F2}");

Sistema de Notificaciones — INotifiable

Creá la interfaz INotifiable con el método Notify(string message) y la propiedad Recipient. Implementá: EmailNotifier, SmsNotifier y ConsoleNotifier. Creá un método estático BroadCast(IEnumerable<INotifiable> notifiers, string msg) que notifique a todos.

public interface INotifiable
{
    string Recipient { get; }
    void Notify(string message);
}

public class EmailNotifier(string email) : INotifiable
{
    public string Recipient => email;
    public void Notify(string message) =>
        Console.WriteLine($"[EMAIL → {email}] {message}");
}

public class SmsNotifier(string phone) : INotifiable
{
    public string Recipient => phone;
    public void Notify(string message) =>
        Console.WriteLine($"[SMS → {phone}] {message}");
}

public class ConsoleNotifier(string name) : INotifiable
{
    public string Recipient => name;
    public void Notify(string message) =>
        Console.WriteLine($"[CONSOLA → {name}] {message}");
}

static void BroadCast(IEnumerable<INotifiable> notifiers, string msg)
{
    foreach (var n in notifiers) n.Notify(msg);
}

// Uso:
List<INotifiable> notifiers = [
    new EmailNotifier("fer@uni.edu.uy"),
    new SmsNotifier("099123456"),
    new ConsoleNotifier("Admin")
];
BroadCast(notifiers, "¡Sistema iniciado!");

Implementación Explícita — Conversión de Temperatura

Creá dos interfaces: ICelsius con GetTemperature() y IFahrenheit también con GetTemperature(). Implementá la clase WeatherStation usando implementación explícita para retornar la temperatura en la escala correcta según la interfaz usada.

public interface ICelsius
{
    double GetTemperature(); // retorna en °C
}

public interface IFahrenheit
{
    double GetTemperature(); // retorna en °F
}

public class WeatherStation(double celsius) : ICelsius, IFahrenheit
{
    // Implementación explícita para ICelsius
    double ICelsius.GetTemperature() => celsius;

    // Implementación explícita para IFahrenheit
    double IFahrenheit.GetTemperature() => celsius * 9.0 / 5.0 + 32;
}

// Uso:
WeatherStation station = new WeatherStation(100);

ICelsius c = station;
Console.WriteLine($"Celsius: {c.GetTemperature()}°C");    // 100°C

IFahrenheit f = station;
Console.WriteLine($"Fahrenheit: {f.GetTemperature()}°F"); // 212°F

Contenedor Genérico con Restricción

Implementá la clase genérica TypedStack<T> (pila LIFO) con los métodos Push(T), T Pop(), T Peek(), int Count y bool IsEmpty. Utilizá List<T> internamente. Probala con int y con string.

public class TypedStack<T>
{
    private readonly List<T> _items = new();

    public int  Count   => _items.Count;
    public bool IsEmpty => _items.Count == 0;

    public void Push(T item) => _items.Add(item);

    public T Pop()
    {
        if (IsEmpty) throw new InvalidOperationException("Pila vacía");
        T top = _items[^1];
        _items.RemoveAt(_items.Count - 1);
        return top;
    }

    public T Peek()
    {
        if (IsEmpty) throw new InvalidOperationException("Pila vacía");
        return _items[^1];
    }
}

// Prueba con int
var stack = new TypedStack<int>();
stack.Push(10); stack.Push(20); stack.Push(30);
Console.WriteLine(stack.Pop()); // 30
Console.WriteLine(stack.Peek()); // 20

// Prueba con string
var strStack = new TypedStack<string>();
strStack.Push("hola"); strStack.Push("mundo");
Console.WriteLine(strStack.Pop()); // mundo

Herencia de Interfaces — Sistema de Pagos

Definí: IPayable con Pay(decimal amount) y Balance. IRefundable extiende IPayable y agrega Refund(decimal amount). Implementá CreditCard : IRefundable y BankTransfer : IPayable (sin refund). Mostrá polimorfismo: una función que recibe IPayable y procesa el pago.

public interface IPayable
{
    decimal Balance { get; }
    void Pay(decimal amount);
}

public interface IRefundable : IPayable
{
    void Refund(decimal amount);
}

public class CreditCard(decimal limit) : IRefundable
{
    public decimal Balance   { get; private set; } = limit;
    public void Pay(decimal a) {
        if (a > Balance) throw new Exception("Saldo insuficiente");
        Balance -= a;
        Console.WriteLine($"[Tarjeta] Pagado ${a}. Saldo: ${Balance}");
    }
    public void Refund(decimal a) {
        Balance += a;
        Console.WriteLine($"[Tarjeta] Reintegro ${a}. Saldo: ${Balance}");
    }
}

public class BankTransfer(decimal funds) : IPayable
{
    public decimal Balance { get; private set; } = funds;
    public void Pay(decimal a) {
        Balance -= a;
        Console.WriteLine($"[Transferencia] Pagado ${a}. Saldo: ${Balance}");
    }
}

static void ProcessPayment(IPayable method, decimal amount) =>
    method.Pay(amount);

// Uso polimórfico:
IPayable card  = new CreditCard(1000m);
IPayable bank  = new BankTransfer(500m);
ProcessPayment(card, 200m);
ProcessPayment(bank, 150m);
if (card is IRefundable r) r.Refund(50m); // solo tarjeta
