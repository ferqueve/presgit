Validación de Formularios

en Bootstrap 5

Aprende a crear formularios con validación robusta y elegante

Grupos 310 y 316

1 ¿Qué es la Validación de Formularios?

La validación de formularios es el proceso de verificar que los datos ingresados por el usuario cumplan con ciertos criterios antes de ser procesados.

Tipos de Validación

Validación del Cliente

  • Se ejecuta en el navegador
  • Respuesta inmediata
  • Mejora la experiencia del usuario
  • No es suficiente por sí sola

Validación del Servidor

  • Se ejecuta en el backend
  • Es imprescindible para seguridad
  • Valida datos antes de guardar
  • Última línea de defensa
Importante: Nunca confíes solo en la validación del cliente. Siempre valida en el servidor.

2 Validación HTML5 Nativa

HTML5 proporciona atributos de validación integrados que los navegadores entienden automáticamente.

Atributos de Validación Principales

Atributo Descripción Ejemplo
required Campo obligatorio <input required>
type Define el tipo de dato esperado <input type="email">
minlength / maxlength Longitud mínima/máxima <input minlength="3">
min / max Valor mínimo/máximo (números) <input type="number" min="1">
pattern Expresión regular personalizada <input pattern="[0-9]{3}"> 
<!-- Ejemplo de validación HTML5 básica -->
<form>
    <input type="text" required minlength="3" placeholder="Nombre">
    <input type="email" required placeholder="Email">
    <input type="number" min="18" max="100" placeholder="Edad">
    <button type="submit">Enviar</button>
</form>

3 Clases de Bootstrap para Validación

Bootstrap proporciona clases CSS para mostrar estados de validación de forma visual y atractiva.

Clases Principales

Estados Válidos

  • .is-valid - Campo válido
  • .valid-feedback - Mensaje de éxito
  • .valid-tooltip - Tooltip de éxito

Estados Inválidos

  • .is-invalid - Campo inválido
  • .invalid-feedback - Mensaje de error
  • .invalid-tooltip - Tooltip de error
<!-- Campo válido -->
<input type="text" class="form-control is-valid" value="Juan Pérez">
<div class="valid-feedback">
    ¡Se ve bien!
</div>

<!-- Campo inválido -->
<input type="email" class="form-control is-invalid" value="correo">
<div class="invalid-feedback">
    Por favor ingresa un email válido.
</div>
Vista previa:
¡Se ve bien!
Por favor ingresa un email válido.

4 Validación con JavaScript

Bootstrap recomienda usar JavaScript para aplicar validación personalizada y deshabilitar la validación nativa del navegador.

Método Recomendado por Bootstrap

<form class="needs-validation" novalidate>
    <div class="mb-3">
        <label for="nombre" class="form-label">Nombre</label>
        <input type="text" class="form-control" id="nombre" required>
        <div class="invalid-feedback">
            Por favor ingresa tu nombre.
        </div>
    </div>
    <button class="btn btn-primary" type="submit">Enviar</button>
</form>
// JavaScript para validación personalizada
const forms = document.querySelectorAll('.needs-validation');

Array.from(forms).forEach(form => {
    form.addEventListener('submit', event => {
        if (!form.checkValidity()) {
            event.preventDefault();
            event.stopPropagation();
        }
        form.classList.add('was-validated');
    }, false);
});
Tip: El atributo novalidate deshabilita la validación nativa del navegador para que podamos usar los estilos de Bootstrap.

6 Validación Personalizada Avanzada

Puedes crear validaciones personalizadas más complejas usando JavaScript.

Ejemplo: Validar que dos campos coincidan

<div class="mb-3">
    <label for="password1">Contraseña</label>
    <input type="password" class="form-control" id="password1" required>
</div>

<div class="mb-3">
    <label for="password2">Confirmar Contraseña</label>
    <input type="password" class="form-control" id="password2" required>
    <div class="invalid-feedback">Las contraseñas no coinciden.</div>
</div>
// Validación personalizada
const password1 = document.getElementById('password1');
const password2 = document.getElementById('password2');

password2.addEventListener('input', function() {
    if (password1.value !== password2.value) {
        password2.setCustomValidity('Las contraseñas no coinciden');
        password2.classList.add('is-invalid');
    } else {
        password2.setCustomValidity('');
        password2.classList.remove('is-invalid');
        password2.classList.add('is-valid');
    }
});

Ejemplo Interactivo:

Las contraseñas coinciden.
Las contraseñas no coinciden.

7 Tooltips vs Feedback Messages

Bootstrap ofrece dos formas de mostrar mensajes de validación:

Feedback (por defecto)

<input class="form-control is-invalid">
<div class="invalid-feedback">
    Mensaje de error
</div>

Ventajas: Siempre visible, más espacio

Este es un mensaje de feedback normal.

Tooltip

<div class="position-relative">
    <input class="form-control is-invalid">
    <div class="invalid-tooltip">
        Mensaje de error
    </div>
</div>

Ventajas: Ahorra espacio, elegante

Este es un tooltip de error.
Recomendación: Usa feedback para formularios importantes y tooltips cuando necesites ahorrar espacio vertical.

8 Mejores Prácticas

Sí hacer:

  • Validar tanto en cliente como en servidor
  • Mostrar mensajes claros y específicos
  • Validar en tiempo real (después del primer intento)
  • Usar íconos para reforzar el estado
  • Marcar campos obligatorios con *
  • Deshabilitar el botón mientras se procesa

No hacer:

  • Confiar solo en validación del cliente
  • Mensajes genéricos como "Error"
  • Validar cada tecla presionada (molesto)
  • Ocultar qué campos fallan
  • Pedir información innecesaria
  • Borrar el formulario tras un error
Seguridad: La validación del cliente es para UX, no para seguridad. Siempre valida y sanitiza en el servidor.
